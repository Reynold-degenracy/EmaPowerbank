# 反馈功能说明

## 功能入口

普通用户登录后，侧边栏会显示与“面板”“日志”同级的“反馈”入口。管理员端也会显示“反馈”入口，并放在“反馈审核”入口上方。

反馈表单包含：

- 必填问题描述，输入框提示为“请简单描述问题现象”
- 可选图片附件，最多 10 张

用户提交前需要二次确认。提交成功后，用户侧只显示“提交成功”，不会显示后端生成的反馈包目录名。

问题反馈被采纳后，会获得一定金额作为奖励。奖励金额由管理员审核时选择，并增加到提交用户的账户余额中；未审核或被拒绝的反馈不会发放奖励。

## 提交接口

前端通过 `POST /api/feedback` 提交反馈。接口要求用户已登录，并使用 `multipart/form-data`。

表单字段：

- `description`：必填，问题描述，提交前后端都会去除首尾空白，最大 5000 字符
- `attachment`：可选，图片附件；可以重复提交该字段，最多 10 张

附件支持常见图片格式：

- PNG
- JPEG / JPG
- GIF
- WebP
- BMP
- AVIF
- HEIC / HEIF

单个附件最大 5 MB。前端会先做格式、数量和大小校验，后端仍会再次校验。

## 保存位置

后端会在项目运行目录下创建 `feedback/`，每次提交保存为一个独立目录：

```text
feedback/feedback-<id>-<timestamp>/
```

示例：

```text
feedback/feedback-fb_3d75a673-5aaf-4c60-9342-472feac83378-2026-06-09T13-38-03-368Z/
```

其中：

- `feedback-` 是固定前缀
- `fb_3d75a673-5aaf-4c60-9342-472feac83378` 是反馈 id，从 `fb_` 开始
- `2026-06-09T13-38-03-368Z` 是提交时间戳

目录名中的时间戳会把 ISO 时间里的 `:` 和 `.` 替换成 `-`，方便作为文件系统路径使用。原始时间戳仍保存在 `feedback.json` 的 `timestamp` 字段里。

## 反馈包内容

每个反馈目录固定包含：

```text
feedback.json
```

如果用户上传了图片，还会包含一个或多个图片文件，文件名同样包含反馈 id、时间戳和序号：

```text
feedback-fb_<uuid>-<timestamp>-01.<ext>
feedback-fb_<uuid>-<timestamp>-02.<ext>
```

`feedback.json` 包含：

- `id`：反馈 id
- `timestamp`：原始 ISO 提交时间
- `packageName`：反馈包目录名
- `user`：提交用户的 id、用户名和角色
- `description`：问题描述文本
- `attachments`：附件元数据数组；没有附件时为空数组
- `review`：审核状态和奖励信息

附件元数据包含：

- `fileName`：后端保存后的附件文件名
- `originalName`：用户上传时的原始文件名
- `mimeType`：附件 MIME 类型
- `size`：附件字节数

`review` 包含：

- `status`：`pending`、`approved` 或 `rejected`
- `rewardAmount`：审核通过时发放的奖励金额，待审核或拒绝时为 `0`
- `reviewedBy`：审核管理员的 id、用户名和角色；未审核时为 `null`
- `reviewedAt`：审核时间；未审核时为 `null`

旧反馈文件如果没有 `review` 字段，会按 `pending` 处理。

## 管理员审核

管理员可以通过侧边栏的“反馈审核”入口打开审核面板。

审核面板支持：

- 按 `待审核`、`已通过`、`已拒绝` 筛选反馈
- 每个状态列表按每页 10 条分页，并显示当前页、总页数、总条数
- 一键导出全部反馈为 CSV，字段为 `user-name`、`user-id`、`description`、`attachments-filenames`、`review-status`
- 查看反馈提交人、问题描述、提交时间、反馈包名和附件信息
- 问题描述使用带标签的白色只读文本框展示，提交时间和反馈包名显示在问题描述下方
- 对图片附件进行横向预览；有附件的卡片会为图片区域预留更宽空间，多张图片超出宽度时可横向滚动
- 拒绝反馈
- 通过反馈并发放奖励金额

通过反馈时，奖励金额必须大于 `0`。界面中的“发放奖励金额”区域提供 `$1.00`、`$3.00`、`$5.00`、`$10.00` 档位，同时保留手动填写能力，手动输入步进为 `$0.10`。

一个反馈只能审核一次。已通过或已拒绝的反馈不能再次通过，也不会重复发放奖励。

审核信息写回该反馈包里的 `feedback.json`。奖励金额会增加到提交用户现有的账户余额中。

## 运行与维护

`feedback/` 是运行时数据目录，已经加入 `.gitignore`，不会进入版本库。

生产环境建议：

- 限制 `feedback/` 的文件权限
- 按需清理、归档或备份反馈包
- 不要把包含用户反馈或附件的目录提交到代码仓库
